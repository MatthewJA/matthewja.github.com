Programming in academia

Sat 31 October 2015
By MatthewJA

My university prides itself on being research-driven. The net result of this is that I know a lot of students pursuing research projects (I've done two myself), mostly in physics. These projects almost invariably require some degree of programming: experimental data needs to be processed, simulations need to be run, and so on. Whether the project involves code the students have written themselves, code written by their supervisors, or some combination thereof, there is a running theme: A lot of that code is really bad.
